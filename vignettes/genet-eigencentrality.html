Generating networks with target eigencentrality

In this vignette, we show how to generate graphs with an overlapping community structure and a target eigencentrality sequence. We draw the target eigencentrality sequence from a power-law distribution; however, users can input any custom target sequence.

library(genet)

First, we set set the number of communities K and types L, and fix a target community structure. Depending on the application of interest, users can input their custom community structure. Here, for simplicity, we will generate a balanced structure via the generate_balanced_interactions function. This function randomly assigns each type to K_each communities and then compute the resulting type interaction matrix. The community memberships are assigned such that: no types have identical assignments; all communities have at least one type; and the resulting graph of type interactions is connected.

K <- 100
L <- 200
ints <- generate_balanced_interactions(K, L, K_each = 5)$interaction_matrix

Next, we draw the target degree centrality sequence from a power-law degree sequence with the following parameters:

x_min <- 0.001
x_max <- 1
alpha <- 1.5

We set the minimum and maximum eigencentrality to 0.001 and 1, and the power law’s exponent to 1.5. To more closely reproduce the power-law distribution, we set the target eigencentralities equal to the \((L + 1)\)-quantiles of the distribution and then adjust the type cardinalities so that the empirical distribution complies with the power law.

target_eigenc <- qpower_law(seq(0, 1, length.out = L), alpha, x_min, x_max, 
                            step = 0.001)
probs <- dpower_law(target_eigenc, alpha, x_min, x_max, step = 0.001)
ratios <- probs / min(probs)
cards <- round(ratios * 2)

Note that we can control the network sparsity through the average expected degree, which can be set via the target_degree argument. We can now generate a random network.

g <- genet(1, ints, cards, dc = F, target_centrality = target_eigenc,
           target_degree = 20)[[1]]

The output of genet is a list of igraph graph objects.

igraph::vcount(g)
#> [1] 156198
igraph::ecount(g)
#> [1] 1560800

We can then use the igraph::eigen_centrality function to extract the empirical eigencentrality distribution of the graph.

eigenc_distr <- round(igraph::eigen_centrality(g, scale = T)$vector, digits = 3)
